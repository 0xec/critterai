CAINav Change Log

Issue Tracking

CAINav v0.3

Fixes

Added a console warning when objects that contain no meshes are assigned to a MeshFilterSource object. (Issue 11)

Added a conditional compilation so scene objects are included in object selection lists for Unity 3.4+. As of this version the pre-compiled libraries still use Unity 3.3- behavior. So this change only effects Unity projects that use the source distribution or custom compiled libraries. (Issue 12)

Removed the use of nullptr in C++ code. (Issue 12, multi-platform support.)

Converted to using <string.h> in all source files. (Issue 13, multi-platform support.)

Baked InvariantCulture into all numeric conversions. (Issue 14, internationalization.)

Updated MeshUtil so it can handle meshes with more than one sub-mesh. (Issue 16)

Fixed defect in NavmeshParams constructor that was resulting in an upper maxPolysPerTile limit of 6.

Core Changes

Updated source to Recast Navigation r312.

Added the org.critterai.nav.PathCorridor class.

Generalized the CrowdCornerData structure, including refactoring its name to CornerData, so it can be used for multiple purposes. It is no longer restricted to a fixed buffer size. (Breaking API Change)

Generalized the PathCorridorData structure so it can be used for multiple purposes. It is no longer restricted to a fixed buffer size.

Added the IsValidPolyRef() method to the NavmeshQuery and U3DNavmeshQuery classes.

Added and new FindPath() overload to the NavmeshQuery and U3DNavmeshQuery classes. It combines the find poly and find path operations into a single interop call.

Added the IsValid() method to the PathCorridor and U3DPathCorridor classes.

Reorganized source to support distribution directly to a Unity Project. The Unity appropriate .NET source now resides in /src/main/Assets/CAI. (Improves multi-platform support.)

Refactored all interop method signatures to make the code base compatible with the iOS target platform. This does not result in any public API changes. (Improves multi-platform support.)

Added Save/Load functionality to the BakedNavmesh and BakedPolyMesh editors. The save file format is .NET serialized byte array. So deserialize the files as a byte array, then create objects using the appropriate byte array constructor.

Unity Extension Changes

Added the org.critterai.nav.u3d.U3DPathCorridor class.

Retired the NavManger class. Most of its functionality was moved to the new NavSource class. (Breaking Change: Will require recreating navigation meshes in Unity.)

Added the org.critterai.nav.u3d.NavGroup structure and updated classes to use it. (Breaking API Change)

Moved the DrawTriMesh() method from the GLUtil to the DebugDraw class.

Added various new debug drawing classes and methods. See the DebugDraw, CrowdAgentDebug, and NavDebug classes.

Added the Vector3Util.GetDistance2D() method.

Implemented performance improvements to all vector SloppyEquals() methods. This change slightly alters the behavior by switching from an AABB inclusion test to a sphere inclusion test.
